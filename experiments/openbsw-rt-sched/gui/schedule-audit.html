Rate-monotonic theory reference

Rate-monotonic (RM) scheduling. Tasks are preemptive, priorities are static, a shorter period gets a higher priority. Tasks are assumed independent — no blocking, no resource sharing. Deadlines are assumed equal to periods (implicit deadlines).

Liu & Layland (1973). For n periodic tasks scheduled under RM, the task set is feasible if ∑(Ci/Ti) ≤ n·(21/n − 1). This bound decreases from 1.000 at n=1 towards ln(2) ≈ 0.693 as n → ∞. It is sufficient: passing the bound proves feasibility; failing it is inconclusive.

Hyperbolic bound (Bini, Buttazzo, Buttazzo 2003). A tighter sufficient test: ∏(ui + 1) ≤ 2 ⇔ feasible under RM. Still sufficient, not necessary — both tests may fail on a task set that is actually feasible, in which case response-time analysis is the next step.

Response-time analysis (Joseph-Pandya 1986). Exact test. For each task i, iterate Rin+1 = Ci + Bi + ∑j ∈ hp(i)⌈Rin/Tj⌉·Cj until fixed point, where hp(i) is the set of tasks with strictly shorter periods on the same context (ties broken by declaration order). The task set is feasible iff every Ri ≤ Di. This is the test to rely on for ASIL-D evidence — both sufficient and necessary under the independent-task model.

Freedom from interference (ISO 26262). When tasks of different criticality levels share a context, the standard requires evidence that a lower-ASIL task cannot impair a higher-ASIL one in time, memory, or resource usage. This tool flags mixed-criticality contexts so you remember to check for timing protection (MPU / timing monitor / freedom-from-interference argument).
