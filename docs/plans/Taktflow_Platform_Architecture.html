What this is
A platform, not a tool. The factory runs across every project in the supply chain — OEM to Tier-1 to Tier-2 — with a consistent pipeline, consistent evidence, and every deviation reviewed and signed off by the right human before it moves forward.
The human contract
Automation generates the evidence. Humans own the decisions. Every CI/CD artifact is pushed to a role-based backlog. The Safety Manager signs safety evidence. The BSW Lead signs ARXML changes. The Test Manager signs test results. Nothing reaches production without a complete sign-off chain.
Why this scales
One factory instance. Multiple projects. Multiple teams. Multiple tiers. The pipeline is identical for every project — only the DBC and requirements change. Evidence quality is consistent across every project the factory touches, not dependent on which engineer set up the toolchain this time.

How a Deviation Gets Resolved
1
CI detects deviation
HIL measures safe state at 11.2ms. Requirement says 10ms. Pipeline does not fail silently — it names the requirement, names the safety goal, shows the delta.
2
Artifact pushed to backlog
Deviation automatically routed to Safety Manager's backlog with full context: measured value, requirement ID, HARA traceability, commit hash, test name.
3
Human reviews and decides
Safety Manager sees the full picture. Three options: Reject — fix required before any progress. Accept with justification — documented deviation. Escalate — push to OEM or change request.
4
Sign-off recorded immutably
Decision committed to git with name, timestamp, and justification. Becomes part of the safety case. Auditor can see every deviation and every human decision made about it.
5
Gate cleared — pipeline continues
Only after all role-based sign-offs are complete does the pipeline allow progression to the next stage — or to production release.
